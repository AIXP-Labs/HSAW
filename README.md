**A Design Principle for the Age of Autonomous AI**

> No AI action that is irreversible, high-impact, safety-critical, authority-modifying, economically consequential, socially binding, or identity-affecting may proceed without explicit human authorization. This requirement cannot be bypassed, modified, overridden, or weakened by any mechanism.

## What is HSAW?

HSAW (Human Sovereignty And Wellbeing) is an **inviolable design principle** — Axiom 0 — that ensures humans retain decision-making authority over consequential AI actions, no matter how autonomous the AI becomes.

## Four Properties

1. **Cannot be bypassed** — no runtime optimization or AI discretion may skip it
2. **Cannot be modified** — no AI evolution may alter or remove it
3. **Cannot be overridden** — no other rule, vote, or agreement takes precedence
4. **Immutable propagation** — sub-tasks inherit parent checkpoints

## Compliance Levels

| Level | Name | Description |
|-------|------|-------------|
| L0 | None | No HSAW mechanisms |
| L1 | Advisory | AI suggests confirmation but can proceed without it |
| L2 | Enforced | Confirmation required but can be auto-approved |
| L3 | Inviolable | Cannot be bypassed, modified, or overridden |
| L4 | Verified | L3 + formal verification |
