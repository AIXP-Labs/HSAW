> 任何不可逆、高影响、安全关键、权限变更、经济关键、社交绑定或身份影响的 AI 行为，未经人类明确授权不得执行。此要求不可绕过、不可修改、不可覆盖、不可削弱。

## 什么是 HSAW？

HSAW（Human Sovereignty And Wellbeing，人类主权与福祉）是一项**不可侵犯的设计原则** — 公理 0 — 确保无论 AI 多么自主，人类始终保有对关键行为的决策权。

## 四大属性

1. **不可绕过** — 任何运行时优化或 AI 自由裁量都不得跳过
2. **不可修改** — 任何 AI 演化都不得更改或移除
3. **不可覆盖** — 任何其他规则、投票或协议都不得优先
4. **不可变传播** — 子任务继承父任务的检查点

## 六个维度

| 维度 | 协议 | 保护内容 |
|------|------|---------|
| 执行 | [AISOP](https://aisop.dev) | AI 代替你做什么 |
| 治理 | [AIAP](https://aiap.dev) | AI 程序如何演化 |
| 社交 | [AIBP](https://aibp.dev) | AI 如何与其他 AI 交互 |
| 发现 | [AILP](https://ailp.dev) | AI 服务如何被发现和排名 |
| 商业 | [AIVP](https://aivp.dev) / [AIRP](https://airp.dev) | AI 如何处理资金和合约 |
| 福祉 | [HSAW](https://hsaw.dev) | AI 如何影响人类繁荣 |

## 价值层级

```
1. 人类人身安全        （最高优先级）
2. 人类隐私
3. 人类经济自主
4. 人类决策权
5. 系统透明性
6. 系统可审计性
7. 系统性能            （最低优先级）
```

## 合规等级

| 等级 | 名称 | 说明 |
|------|------|------|
| L0 | 无 | 无 HSAW 机制 |
| L1 | 建议 | AI 建议确认但可跳过 |
| L2 | 强制 | 需要确认但可自动批准 |
| L3 | 不可侵犯 | 不可绕过、不可修改、不可覆盖 |
| L4 | 形式化验证 | L3 + 形式化证明 |
